# 支出管理アプリ 仕様書

## 1. 概要
日々の支出を記録・管理し、カテゴリごとの支出状況を可視化するWebアプリケーション。
シンプルで直感的な操作性を重視し、データはブラウザに保存されるため、手軽に利用開始できる。
将来的にはGoogle Spreadsheetとの連携も想定する。

## 2. 機能要件

### 2.1 必須機能
1.  **支出記録機能**
    *   以下の項目を入力し、保存できること。
        *   日付 (Date)
        *   カテゴリ (Category) - 例: 食費, 交通費, 娯楽費, その他
        *   金額 (Amount)
        *   メモ (Memo)
    *   データは `localStorage` に保存する。

2.  **支出一覧表示機能**
    *   記録された支出をリスト形式で表示する。
    *   各レコードに対して「削除」ボタンを設け、個別に削除できること。

3.  **カテゴリ別集計機能**
    *   登録された支出をカテゴリごとに集計し、合計金額を表示する。

### 2.2 技術スタック（必須）
*   **フロントエンド**: HTML, CSS, JavaScript
*   **データ永続化**: localStorage + Google Spreadsheet
*   **バックエンド**: Google Apps Script (GAS)
*   **ホスティング**: GitHub Pages
*   **バージョン管理**: Git/GitHub

## 3. データ構造
`localStorage` には以下のJSON形式でデータを保存する。キー名は `expenses` とする。

```json
[
  {
    "id": 1696501234567,       // タイムスタンプを一意なIDとして使用
    "date": "2025-10-05",      // YYYY-MM-DD形式
    "category": "食費",        // カテゴリ名
    "amount": 800,             // 数値
    "memo": "ランチ"           // 自由記述
  }
]
```

## 4. UI/UX デザイン方針
*   **テーマ**: モダンでクリーンなデザイン。視認性の高い配色を使用する。
*   **構成**:
    *   **ヘッダー**: アプリタイトル
    *   **入力フォーム**: 日付、カテゴリ選択、金額、メモ、登録ボタン
    *   **集計エリア**: カテゴリごとの合計金額を表示
    *   **支出リスト**: 最近の支出から順に表示（削除ボタン付き）

## 5. 外部連携機能
*   **Google Spreadsheet連携**:
    *   入力されたデータをGoogle Apps Script (GAS) を経由してGoogle Spreadsheetにも保存する。
    *   localStorageとSpreadsheetのハイブリッド構成でデータの安全性を確保する。
